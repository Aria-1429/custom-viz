# Custom Viz Waterfall Chart

Splunk Dashboard Studio 用のカスタムビジュアライゼーション。**ウォーターフォール（滝／ブリッジ）チャート**で、開始値からの増減の積み上げが最終的な合計へどう届くかを階段状のバーで可視化する。Splunk 標準ビジュアライゼーション（column / bar / line / pie 等）には存在しないチャート。

売上→費用→利益のブリッジ、エラー件数の要因分解、在庫・ライセンス数の増減内訳、キャパシティの変動要因など「増減の内訳を見せる」用途全般に使える。

## 特徴

- 増加＝緑・減少＝赤・合計/開始＝青の階段状バー（色は編集パネルで変更可、増減の色の入れ替えも可）
- バー間を破線コネクタで接続、符号付き値ラベル、Y軸目盛・ゼロ基準線
- 末尾に「合計」バーを自動追加（データが total 行で終わる場合は追加しない）
- 種別列（`total` / `start` / `合計` / `開始` など）でバーの種別を行単位に指定可能。種別トークンだけの列は**自動検出**
- 「値を累計値として解釈」モード：累計スナップショット系列（例: 月末在庫）から増減を自動計算
- ラベル・値・種別フィールドをドロップダウンで選択（editor.columnSelector、DOS 文字列を自前解決）
- 負の累計、マルチバリューセル救済、rows / columns 両形式対応
- コンテナ実寸へ自動フィット。小さいパネルでは値ラベル → X軸ラベル → Y軸目盛の順に段階退避。長い項目名は回転＋切り詰め
- バー伸長アニメーション（左から順に、オフ可）
- ライト／ダークテーマ対応、編集パネルは日本語ラベル

## データ仕様

| 列 | 内容 |
| --- | --- |
| ラベル列（既定: 第1列） | ステップ名 |
| 値列（既定: 数値を含む最終列） | 増減値（正=増加、負=減少） |
| 種別列（任意） | `total`/`合計`/`小計` → その時点の累計を 0 からの絶対バーで描画。`start`/`開始` → 値を絶対値として累計をリセット。空欄 → 通常の増減 |

「値を累計値として解釈」ON のときは各行の値を累計とみなし、増減＝前行との差分を自動計算する（第1行は開始バー）。

## サンプル SPL

増減データ（2列・合計バー自動追加）:

```spl
| makeresults format=csv data="項目,増減
新規契約,420
アップグレード,180
解約,-260
ダウングレード,-90
再契約,60"
```

種別列つき（利益ブリッジ型）:

```spl
| makeresults format=csv data="項目,金額,種別
期首残高,5000,start
新規売上,3200,
返品,-450,
値引き,-380,
中間累計,,subtotal
追加受注,900,
期末残高,,total"
```

累計値モード（「値を累計値として解釈」を ON にする）:

```spl
| makeresults format=csv data="月,ライセンス数
1月,1200
2月,1350
3月,1280
4月,1490
5月,1470
6月,1620"
```

## 開発

```bash
yarn install
yarn build     # dist/custom_viz_waterfall_chart/visualization.js
yarn verify    # happy-dom によるローカル検証（Splunk 実機なし）
yarn package   # dist/custom_viz_waterfall_chart-<ver>-<hash>.spl
```

## デプロイ

1. バージョンを上げて `yarn build && yarn package`
2. Splunk Web「App をファイルからインストール」で **「App のアップグレード」にチェック**して `.spl` をアップロード
3. `https://<host>:8000/en-US/_bump` で Bump version
4. ブラウザをハードリロード（Ctrl+Shift+R）

Dashboard Studio の JSON では `"type": "custom_viz_waterfall_chart.custom_viz_waterfall_chart"` を指定する。
