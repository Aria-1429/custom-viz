# Custom Viz Funnel Leak（アニメ付きファネル×リーク図）

![表示例](examples/example.png)

Splunk Dashboard Studio 向けのカスタムビジュアライゼーション。
**ファネルの各段で「どこで・どれだけ離脱したか」** を、
下方向に流れる通過フローと、左右にこぼれ落ちる離脱（リーク）フローのアニメーションで可視化します。

標準の「円」「横棒」「サンキー」では、各段の通過数は出せても
「離脱がどの段でどれだけ発生したか」を直感的に見せることはできません。
本 viz は通過分を次段へ流し、離脱分を側面に「こぼす」ことで、
コンバージョン分析・ログイン→購入導線・攻撃チェーンの生存分析などをひと目で読めるようにします。

## 特徴

- **段ごとのバー**：幅が件数に比例（先頭段＝最大を基準に相対化）。上→下に段が積み上がるファネル形。
- **通過フロー**：段 i→i+1 を漏斗の首で結び、流れる粒子で「量」を体感（既定は粒子のみ。
  帯の塗りは「フローの帯を表示」でオンにできる）。
- **離脱（リーク）フロー**：`count[i] − count[i+1]` を、各段バー中央の高さから左右交互に
  こぼれ落ちる粒子として描画し、終端に向かってフェードアウト。離脱量に応じて粒子数が変化。
- **注記の統一配置**：段階名と離脱数（−N）は左の枠外に左寄せで統一、右側に件数・通過率を
  日本語で併記。
- **通過率カラースケール**（任意）：段の色を通過率で低→高に補間（例：低い通過率＝赤、高い＝緑）。
- **ホバーで段をハイライト**：関連する前後のフロー・粒子だけを強調。
- **オートフィット**：コンテナ実寸を ResizeObserver で測って自動レイアウト。収まる限りスクロールを
  出さず領域いっぱいに広げ、段数が多く収まらない場合のみ縦スクロールする。
- **ライト/ダークテーマ対応**、外部通信なし（完全オフライン）。
- 編集画面のオプションはすべて日本語ラベル。

## データ仕様

- **1 行 = 1 ステップ**。**行の順序がそのまま上→下の段順**になります（SPL 側で段順に並べてください）。
- **段階名フィールド**：文字列（既定は第 1 列）。
- **件数フィールド**：その段に到達した件数（既定は最終列）。カンマ・空白付きの数値も正規化。
- フィールドは編集画面の「データフィールド」でドロップダウン選択できます（未指定なら既定列）。
- マルチバリューセルは全列のトークン数が一致する場合のみ平行展開して救済。

## 編集画面オプション

| セクション | 項目 |
| --- | --- |
| データフィールド | ステップ／件数フィールドの選択 |
| 色 | 通過フロー色・離脱色・通過率カラースケール（低/高色） |
| アニメーション | 周期（秒、0 で停止）・粒子密度・フローの帯を表示（粒子の下地、既定 OFF） |
| レイアウト | バーの太さ・段の間隔・角の丸み・リーク帯幅（いずれも 0 で自動） |
| 表示 | 段階名／件数／通過率／離脱数／総合通過率ヘッダー・ラベルサイズ・ホバー強調・サマリーヘッダー |
| デバッグ | options の生値オーバーレイ |

## 開発コマンド

```bash
yarn install
yarn build        # dist/custom_viz_funnel_leak/visualization.js を生成
yarn verify       # happy-dom によるローカル検証（Splunk 実機不要）
yarn package      # dist/*.spl を生成
```

## デプロイ（アンインストール・再起動なし）

1. `npm version <patch|minor> --no-git-tag-version` でバージョンを上げ、`package/app/app.conf` の version も同期。
2. `yarn build && yarn package` で新しい `.spl` を生成。
3. Splunk Web「Install app from file」で **"Upgrade app"（上書き）にチェック**してアップロード。
4. `https://<host>:8000/en-US/_bump` で **Bump version** → ブラウザをハードリロード（Ctrl+Shift+R）。

## サンプル SPL

### 1. 購入ファネル（基本）

```spl
| makeresults format=csv data="step,users
サイト訪問,10000
カート追加,6000
決済開始,3500
購入完了,1400"
```

「サイト訪問 → カート追加」で 4,000 離脱（通過 60%）、総合通過率 14% が視覚化されます。

### 2. ログイン導線

```spl
| makeresults format=csv data="phase,count
ログインページ表示,52000
認証情報入力,38000
多要素認証,29500
ログイン成功,27100"
```

### 3. 攻撃チェーンの生存分析（セキュリティ）

```spl
| makeresults format=csv data="stage,events
偵察スキャン,8400
初期侵入試行,2100
権限昇格,640
横展開,180
データ持ち出し,42"
```

各段でどれだけブロック（離脱）できたかがリークとして見えます。

### 4. 実データ例（stats で段ごとに集計してから段順に並べる）

```spl
index=web sourcetype=access_combined
| eval step=case(
    uri_path="/",              "訪問",
    uri_path="/cart",          "カート",
    uri_path="/checkout",      "決済",
    uri_path="/complete",      "完了")
| stats dc(session_id) as users by step
| eval order=case(step="訪問",1, step="カート",2, step="決済",3, step="完了",4)
| sort order
| fields step users
```

> 注意：`makemv | mvexpand | rex` のチェーンは実機で不発になりやすいので、
> `makeresults format=csv` 形式を推奨します。
